# Installation

## One-line install

```bash
curl -fsSL https://vyprai.github.io/loka/install.sh | bash
```

No sudo required. Installs to `~/.loka/bin`.

On **macOS**: installs all binaries natively. LOKA uses the **Apple Virtualization Framework** to run microVMs directly on your Mac -- no Docker, no Lima, no nested virtualization required.

On **Linux**: installs all binaries (`lokad`, `loka`, `loka-worker`, `loka-supervisor`) and configures **Cloud Hypervisor** with KVM for running microVMs.

### Options

```bash
# Specific version
LOKA_VERSION=v0.1.0 curl -fsSL https://vyprai.github.io/loka/install.sh | bash

# Custom install directory
LOKA_INSTALL_DIR=/opt/loka/bin curl -fsSL https://vyprai.github.io/loka/install.sh | bash

# Custom data directory
LOKA_DATA_DIR=/opt/loka/data curl -fsSL https://vyprai.github.io/loka/install.sh | bash
```

## PATH setup

Add `~/.local/bin` to your PATH if not already present:

```bash
export PATH="$HOME/.local/bin:$PATH"
```

## Verify

```bash
loka version
```

## Start

```bash
loka setup local                              # Start the control plane + worker
loka image pull python:3.12-slim              # Pull an image
loka session create --image python:3.12-slim  # Create a session
loka exec <id> -- echo "it works"             # Run a command
```

## Install the SDKs

```bash
pip install loka-sdk
```

```bash
npm install @vypr-ai/loka-sdk
```

## Prerequisites

| Platform | Requirement | Why |
|----------|-------------|-----|
| **macOS** | Apple Silicon or Intel with Hypervisor.framework | Apple Virtualization Framework requires hardware virtualization |
| **Linux** | `/dev/kvm` | Cloud Hypervisor requires KVM |

No Docker daemon is required. LOKA uses [crane](https://github.com/google/go-containerregistry) to pull images directly from registries.

## Build from source

```bash
git clone https://github.com/vyprai/loka && cd loka
make install    # Builds and installs to ~/.loka/bin
```

### Binaries

| Binary | What it does |
|--------|-------------|
| `lokad` | Control plane -- API server, scheduler, session manager |
| `loka-worker` | Worker agent -- manages microVMs |
| `loka-supervisor` | Runs inside VM as PID 1 -- command proxy, sandbox |
| `loka` | CLI client |

## Alternative: Docker

```bash
docker build -f deploy/docker/Dockerfile.controlplane -t loka-cp .
docker build -f deploy/docker/Dockerfile.worker -t loka-worker .
```

## Alternative: Systemd

```bash
sudo cp deploy/systemd/lokad.service /etc/systemd/system/
sudo systemctl enable --now lokad
```

## Configuration

### Control Plane

`/etc/loka/controlplane.yaml` (or set via `LOKA_CONFIG` env var / `lokad --config <path>` / `lokad -c <path>`)

```yaml
role: all                 # "all" (CP + embedded worker) or "controlplane" (CP only)
mode: single              # "single" or "ha"
listen_addr: ":6840"      # REST API (fallback for CLI and SDKs)
grpc_addr: ":6841"        # gRPC (primary -- CLI, SDKs, and workers default to this)

database:
  driver: sqlite          # "sqlite" or "postgres"
  dsn: "/var/loka/loka.db"

coordinator:
  type: local             # "local" or "raft"

objectstore:
  type: local
  path: "/var/loka/artifacts"

scheduler:
  strategy: spread        # "spread" or "binpack"

auth:
  api_key: ""             # Set to require Bearer token

logging:
  format: text            # "text" or "json"
  level: info
```

### Environment Variables

| Variable | Default |
|----------|---------|
| `LOKA_CONFIG` | `/etc/loka/controlplane.yaml` |
| `LOKA_KERNEL_PATH` | `/var/loka/kernel/vmlinux` |
| `LOKA_ROOTFS_PATH` | `/var/loka/rootfs/rootfs.ext4` |

## macOS

On macOS, LOKA runs microVMs natively using the **Apple Virtualization Framework**. No Docker, Lima, or nested virtualization is required.

```bash
curl -fsSL https://vyprai.github.io/loka/install.sh | bash
```

After installation:

```bash
loka setup local                              # Starts lokad with embedded worker
loka image pull python:3.12-slim
loka session create --image python:3.12-slim
loka exec <id> -- python3 -c "print(42)"
loka space down                               # Stops lokad
```

## Linux

On Linux, LOKA uses **Cloud Hypervisor** with KVM to run microVMs. Ensure `/dev/kvm` is available.

```bash
curl -fsSL https://vyprai.github.io/loka/install.sh | bash
```

After installation:

```bash
loka setup local
loka image pull python:3.12-slim
loka session create --image python:3.12-slim
loka exec <id> -- python3 -c "print(42)"
loka space down
```
