# Volumes

Volumes provide persistent storage for sessions and services. Data in a volume survives session restarts, redeployments, and can be shared across components.

## Create a volume

```bash
loka volume create my-data --size 10G
```

## List volumes

```bash
loka volume list
```

## Inspect a volume

```bash
loka volume inspect my-data
```

Shows volume details including size, usage, attached services, and sync status.

## Delete a volume

```bash
loka volume delete my-data
```

## Mount a volume in a service

In `loka.yaml`:

```yaml
name: my-app
components:
  api:
    image: python:3.12-slim
    command: uvicorn main:app --host 0.0.0.0 --port 8000
    port: 8000
    mounts:
      - volume: my-data
        path: /data
      - volume: db-data
        path: /var/lib/postgresql/data
        read_only: true
```

Volumes are created automatically if they do not already exist.

## Mount a volume in a session

```bash
loka session create --image python:3.12-slim --volume my-data:/data
```

## Cross-worker sync

When a service is rescheduled to a different worker, its volumes are automatically synced to the new host. LOKA uses an internal volume sync mechanism to ensure data availability across the cluster.

The sync process:

1. When a workload is scheduled on a worker, the scheduler checks if the required volumes are present.
2. If a volume is on a different worker, the volume sync copies the data to the target worker.
3. The workload starts once the volume is available locally.

For large volumes, this may take time. The scheduler prefers workers that already have the required volumes to minimize sync overhead.

## Storage mounts (cloud)

For sessions, you can also mount cloud storage (S3, GCS) directly:

```bash
loka session create --image python:3.12-slim \
  --mount s3://bucket@/data?access_key_id=AKIA...&secret_access_key=...
```

Sync data between the VM and cloud storage:

```bash
loka session sync <id> /data --direction push   # VM --> cloud
loka session sync <id> /data --direction pull   # cloud --> VM
```
