# Deploy Locally

```bash
loka setup local --name dev
```

This starts `lokad` in the background with an embedded worker, SQLite, and local storage. It uses `role: all` (the default), which runs the control plane and an embedded worker together in one process.

On **macOS**, microVMs run natively using the **Apple Virtualization Framework**. On **Linux**, they use **Cloud Hypervisor** with KVM. No Docker daemon is required.

## Options

| Flag | Default | Description |
|------|---------|-------------|
| `--name` | `local` | Space name |
| `-f, --foreground` | `false` | Run in foreground (attached to terminal) |

## What it creates

| Component | Details |
|-----------|---------|
| Control plane (REST) | `http://localhost:6840` |
| Control plane (gRPC) | `localhost:6841` (primary -- used by CLI and SDKs) |
| Database | SQLite |
| Workers | 1 (embedded) |

## Enable DNS

After starting, enable `.loka` local domain resolution:

```bash
loka dns enable
```

This configures your system to resolve `*.loka` domains to the local control plane. Services become accessible at `http://<service-name>.loka`.

## Deploy an app

```bash
loka deploy nginx:latest --name my-app
# Access at: http://my-app.loka (with DNS enabled)
```

## Declarative setup

You can also use `setup apply` with `provider: local` as an equivalent:

```yaml
name: dev
provider: local
```

```bash
loka setup apply dev.yml
```

## Stop

```bash
loka space down              # Stop the active space
loka space down dev          # Stop a specific space
```

## Destroy

```bash
loka space down dev --destroy
```
